# 朗読動画作成ツール 仕様書 草案

## 1\. アプリ概要

本アプリは、ユーザーが用意した**朗読音声**に対して、**字幕・背景画像・BGM / 環境音**を連動させ、朗読動画を生成するデスクトップアプリである。

オープニング、タイトル画面、作者クレジット、エンディングなどはユーザー側で別途制作・編集する想定とし、本アプリでは扱わない。

## 2\. 目的

朗読動画制作における以下の作業を簡略化する。

- 朗読音声の並び順管理
- 音声ごとの字幕設定
- 音声区間ごとの背景画像指定
- 任意区間へのBGM・環境音配置
- 字幕付き動画の生成

主な利用者は、朗読・怪談・小説読み上げ・ASMR系の動画投稿者、Vtuber、個人クリエイターを想定。

## 3\. 対象範囲

### 3.1 対象とする機能

- 朗読音声ファイルの読み込み
- 字幕テキストの入力・編集
- 音声ごとの無音間隔設定
- 背景画像の適用範囲設定
- BGM・環境音の適用範囲設定
- 字幕スタイル設定
- 動画書き出し
- プロジェクト保存・再読み込み

### 3.2 対象外とする機能

- タイトル映像生成
- 作者クレジット映像生成
- オープニング映像生成
- エンディング映像生成
- 本格的な動画編集
- 音声収録スタジオ機能
- 高度な音声修復

ただし、将来的な拡張として、簡易録音・波形編集・既存動画との結合は検討対象とする。

## 4\. 基本方針

### 4.1 001・002・003 の固定意味を廃止する

公開版では、ファイル名の番号に固定の意味を持たせない。

現行処理では、`001`をタイトル音声、`002`をクレジット音声として探し、`003`をセクションまたは本文開始として扱う構造がある。

公開版ではこれを廃止し、すべての朗読音声を同一の「朗読クリップ」として扱う。

### 4.2 内部IDとファイル名を分離する

- ユーザーのファイル名は自由とする。
  - 例
    - voice_001.wav
    - chapter1_take3.mp3
    - 怪談_前半.m4a
    - 録音_2026-06-30.wav

- アプリ内部では、読み込み順に以下のようなIDを付与する。
  - clip_0001
  - clip_0002
  - clip_0003

字幕、背景画像、BGMの範囲指定は、この内部IDに紐づける。

### 4.3 ユーザーにフォルダ構成を強制しない

内部的にはプロジェクトフォルダを作成して管理してもよいが、ユーザーには以下の操作だけを見せる。

- 音声を追加・削除・ファイルの再指定
- 画像を追加・削除・ファイルの再指定
- BGMを追加・削除・ファイルの再指定
- 字幕を入力
- 動画を書き出す

現行 GUI は `音声ファイル/`、`背景画像/`、`字幕.json` の存在を前提として読み込むが、公開版ではこの構成をユーザーに要求しない。

フォルダから一括で選択できるようにするほか、ドラッグ＆ドロップで対応させることができるようにもする。

### 4.4 素材管理方針

素材追加時、アプリは元ファイルをプロジェクトフォルダ内へコピーする。
project.json には、プロジェクトフォルダからの相対パスを保存する。

ただし、上級設定として「元ファイルを参照するだけ」も選択可能とする。

## 5\. 想定ワークフロー

### 5.1 新規プロジェクト作成

1. アプリを起動する
2. 「新規プロジェクト」を選択する
3. 保存先フォルダを指定する
4. 音声ファイルを追加する
5. 必要に応じて並び順を変更する
6. 各音声に字幕を入力する
7. 背景画像を追加する
8. 画像の適用範囲を指定する（例：画像Aはclip_0001~clip_0003再生中など）
9. BGM / 環境音を追加する
10. BGMの適用範囲と音量を指定する（例：BGM_Aはclip_0001~clip_0005再生中など）
11. プレビュー確認
12. 動画生成

### 5.2 既存プロジェクト再編集

1. ユーザーがプロジェクトフォルダを選択した際、`.rvproj` または `project.json` を開く
2. 前回の音声・字幕・画像・BGM設定を復元する
3. 必要な箇所を修正する
4. 再書き出しする

## 6\. 入力素材仕様

### 6.1 音声ファイル

#### 対応形式

- wav
- mp3
- m4a
- aac
- ogg
- flac

#### 内部変換形式

読み込み時または生成時に、FFmpegで以下へ正規化する。

- WAV
- PCM 16bit
- 48kHz
- stereo

理由は、生身の録音音声では録音環境によって形式が大きくばらつくため。現行処理では `wave.open()` によって WAV を直接扱い、本文音声を連結しているため、公開版ではこの前処理が必須になる。

#### 音声クリップ情報

各音声は以下の情報を持つ。

```json
{
  "id": "clip_0001",
  "file_path": "audio/voice_0001.wav",
  "display_name": "怪談_前半.wav",
  "subtitle": "秋の夜、白輪の眩き月が雲に濡れるように出ていた。","gap_after": 0.5,
  "volume": 1.0,
  "enabled": true
}
```

### 6.2 字幕

#### 字幕の基本単位

字幕は、原則として**音声クリップ1件につき1字幕**とする。

- clip_0001 → 字幕1
- clip_0002 → 字幕2
- clip_0003 → 字幕3

初期版では、音声途中で字幕を複数回切り替える機能は必須としない。

#### 字幕入力方法

- 画面上で直接入力
- JSONから読み込み
- CSVから読み込み
- テキストファイルから一括読み込み

#### 字幕スタイル

最低限、以下を設定可能にする。

- フォント
- 文字サイズ
- 文字色
- 縁取り色
- 影の有無
- 字幕背景ボックスの有無（色や透過の設定も含む）
- 字幕表示位置
- 最大行幅
- 行間

現行 generator.py には、字幕フォントサイズ、下余白、ボックス余白、最大幅、行間、文字色、影色、ボックス色などの固定値が定義されているため、公開版ではこれらを設定項目化するのが自然です。

### 6.3 背景画像

#### 対応形式

- png
- jpg
- jpeg
- webp

#### 表示方法

画像は動画サイズに合わせて自動的にトリミング・拡大縮小する。

- 基本方式: cover
- 画面比率: 16:9
- 既定解像度: 1920 x 1080

現行 generator.py でも背景画像を `fit_cover()` によって動画サイズへ合わせる処理が存在するため、この方針は継続可能です。

#### 背景画像設定項目

- 有効 / 無効
- 開始クリップ
- 終了クリップ

#### 背景画像の適用範囲

背景画像は、音声クリップ単位で適用範囲を指定する。

```json
{
  "image_id": "bg_0001",
  "file_path": "images/forest.png",
  "start_clip_id": "clip_0001",
  "end_clip_id": "clip_0005",
  "enabled": true,
  "transition": {
    "type": "fade",
    "duration": 0.8
  }
}
```

現行処理でも、背景画像は `start` と `end` の音声番号範囲に基づいて選択され、背景切替時にフェードを入れる構造になっています。

公開版では、番号ではなく `clip_id` 基準に変更する。

### 6.4 BGM / 環境音

#### 対応形式

- wav
- mp3
- m4a
- aac
- ogg
- flac

現行 GUI でも BGM フォルダ内の `wav`、`mp3`、`ogg`、`m4a` を検出する構成があります。

#### BGM設定項目

- 有効 / 無効
- ファイル名
- 開始クリップ
- 終了クリップ
- 音量
- フェードイン秒数
- フェードアウト秒数
- ループ有無

#### BGMデータ例

```json
{
  "id": "bgm_0001",
  "file_path": "bgm/rain.mp3",
  "start_clip_id": "clip_0001",
  "end_clip_id": "clip_0012",
  "volume": 0.12,
  "fade_in": 1.5,
  "fade_out": 3.0,
  "loop": true,
  "enabled": true
}
```

現行 generator.py では、FFmpeg の `atrim`、`afade`、`adelay`、`volume`、`amix` を組み合わせてBGMを合成しています。

この設計は継続し、UIとデータ構造のみ汎用化する。

- 映像コーデック:H.264
- 音声コーデック: AAC
- 既定解像度: 1920 x 1080
- 既定FPS: 30

現行 generator.py でも `WIDTH = 1920`、`HEIGHT = 1080`、`FPS = 30` が定数として定義されています。

公開版では、初期値として残し、将来的に設定化する。（Youtubeショート動画サイズを用意しても良いかも）

### 出力ファイル名

- 初期値:reading_video.mp4

ユーザーが任意に変更可能とする。

## 10\. 生成処理仕様

### 10.1 基本処理順

1. プロジェクト設定を検証
2. 音声ファイルを内部形式へ変換
3. 音声クリップを順番に連結
4. 各クリップの開始時刻・終了時刻を算出
5. 背景画像タイムラインを生成
6. 字幕タイムラインを生成
7. BGM / 環境音をタイムラインに配置
8. 映像フレームを書き出し
9. 音声と映像を合成
10. MP4として出力
11. 一時ファイルを削除

### 10.2 タイムライン生成

各音声クリップから以下を算出する。

```json
{
  "clip_id": "clip_0001",
  "start": 0.0,
  "end": 8.24,
  "subtitle": "秋の夜、白輪の眩き月が雲に濡れるように出ていた。",
  "background_id": "bg_0001"
}
```

現行 generator.py でも、本文音声を順に読み込み、`index`、`start`、`end` を持つ `timeline` を生成しています。

公開版では、`index` を `clip_id` に置き換える。

### 10.3 字幕表示

初期版では、音声クリップの再生中だけ字幕を表示する。

- 表示開始: クリップ開始時刻
- 表示終了: クリップ終了時刻

`gap_after` の無音中は、既定では字幕を消す。
プロジェクト全体設定で、「次のクリップ開始まで字幕を残す」も選択可能にする。

### 10.4 背景画像表示

背景画像は、現在再生中の音声クリップに対応する画像を表示する。

- 背景画像が1枚も登録されていない場合:
  - 初期版では黒背景で書き出し可能とする
  - 警告ログを表示する

- 背景画像が登録されているが、範囲未設定の区間がある場合:
  - 直前の背景画像を継続する
  - 直前画像がない場合は黒背景にする

### 10.5 BGM合成

BGMは、指定された開始クリップの開始時刻から、終了クリップの終了時刻まで再生する。

- 開始位置 = start_clip_id の start
- 終了位置 = end_clip_id の end

BGM素材が指定範囲より短い場合は、設定に応じて以下のどちらかを行う。

- ループする
- 不足分は無音にする

## 11\. エラー仕様

ユーザー向けには、Python例外をそのまま表示しない。
以下のような日本語メッセージに変換する。

- 音声ファイルを読み込めませんでした。
- 画像ファイルを読み込めませんでした。
- 背景画像の適用範囲が不正です。
- BGMの開始位置が終了位置より後になっています。
- 動画の書き出しに失敗しました。
- FFmpegが見つかりません。
- 出力先フォルダに書き込み権限がありません。

- 字幕が空のクリップがある場合、警告を表示する。
  - ただし、ユーザーが許可した場合は字幕なしクリップとして書き出し可能とする。

## 15\. 開発優先順位

### 第1段階：現行コードの整理

- generator.py からタイトル・クレジット・セクション処理を外す
- run_generation() を朗読本文専用にする
- 音声番号ではなく clip_id を受け取る形にする
- project.json を定義する

### 第2段階：GUIの汎用化

- 001開始前提を削除
- 音声ファイルを自由追加できるようにする
- 字幕をGUI上で直接編集できるようにする
- 背景画像とBGMをclip_id範囲で指定する

## 16\. 仕様上の結論

このアプリは、**動画編集ソフトではなく、朗読動画の素材同期ツール**として定義するのがよい。

核になる価値は以下。

- 朗読音声を並べる
- 字幕を対応させる
- 背景画像を範囲指定する
- BGMを範囲指定する
- 一括でMP4にする

そのため、公開版の設計では以下を徹底する。

- オープニングは作らない
- タイトルは作らない
- クレジットは作らない
- 音声番号に意味を持たせない
- 朗読本体の同期に集中する

## 17. 初期版で保証する操作

- 音声を追加できる
- 音声を並び替えできる
- 音声ごとに字幕を入力できる
- 音声ごとに後続無音秒数を設定できる
- 背景画像を追加できる
- 背景画像をクリップ範囲に割り当てられる
- BGMを追加できる
- BGMをクリップ範囲に割り当てられる
- MP4を書き出せる
- プロジェクトを保存・再読み込みできる

## 18. 初期版では保証しない操作

- アプリ内録音
- 波形編集
- 字幕の音声途中切り替え
- 動画素材の読み込み
- 複数解像度の詳細設定
- 複数字幕レイヤー
- エフェクト編集
